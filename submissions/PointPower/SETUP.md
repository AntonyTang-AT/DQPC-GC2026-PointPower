# PointPower — Reproduction Guide (Enhancement Only)

This package plus the official UVG-CWI-DQPC dataset reproduces **inference**; with train CG+HE pairs you can also reproduce **PD-LTS fine-tuning**.

Data paths are relative to `$GC2026_ROOT` — see [`data/DATA_LAYOUT.md`](data/DATA_LAYOUT.md).

---

## Official submission channel

Per [UVG-CWI/submissions](https://github.com/UVG-CWI/submissions):

1. **Fork** https://github.com/UVG-CWI/submissions and clone your fork.
2. Copy this package to `submissions/PointPower/` (team directory name = **PointPower**).
3. Ensure `submissions/PointPower/README.md`, `src/`, and `requirements.txt` are present.
4. **Do not commit** input `.ply`/`.bag` datasets or output ENH `.ply` files.
5. Open a **Pull Request** to `UVG-CWI/submissions:main` (use the repo PR template).

Organizers run your `src/` scripts on official data and rank by Chamfer Distance (lower is better).

## 2. Data you must provide (relative to `$GC2026_ROOT`)

### Inference only

```
data/raw/UVG-CWI-DQPC/<Sequence>/consumer-grade_capture_system/CG/15fps/*.ply
```

### PD-LTS fine-tuning / evaluation (CG + HE)

```
data/raw/UVG-CWI-DQPC/<Sequence>/consumer-grade_capture_system/CG/15fps/*.ply
data/raw/UVG-CWI-DQPC/<Sequence>/high-end_capture_system/HE/15fps/*.ply
```

**Train / val sequence** names are read from `data/raw/.../train|val/` or `data/splits/val/<Seq>/` marker directories, then written to `data/splits/split.json` and `data/processed/runtime_gate.json`.

## 3. Directory layout (relative to `$GC2026_ROOT`)

```
./                          ← root after `export GC2026_ROOT=./workspace`
data/raw/UVG-CWI-DQPC/      ← official dataset
data/processed/             ← generated by generate_pair_lists.sh
code/PD-LTS/                ← download_pdlts.sh
code/SuperPC/               ← download_pretrained.sh
code/Metric/                ← download_metric.sh (training / evaluation)
output/                     ← inference and training outputs
```

The submission folder can live anywhere; it is independent of `GC2026_ROOT`.

## 5. Installation

```bash
tar xzf GC2026_Team_EnhancementOnly.tar.gz
cd GC2026_Team_EnhancementOnly
export GC2026_ROOT=../workspace          # directory containing data/raw/

# 1) Place train/val markers per data/splits/README.md
# 2) Auto-discover sequences and write config
bash data/generate_pair_lists.sh

conda create -n superpc python=3.9 -y && conda activate superpc
pip install -r requirements.txt
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu128

bash src/download_pdlts.sh
bash src/download_pretrained.sh
bash src/setup_pdlts_deps.sh
```

## 6. Inference and evaluation

**PD-LTS weight selection**

1. If `output/pdlts_finetune_uvg/run_*/DenoiseFlow-light-UVG-finetune.ckpt` exists → use fine-tuned checkpoint.
2. Otherwise → use bundled `models/DenoiseFlow-light-UVG-finetune.ckpt`.

```bash
bash src/run_smoke.sh          # 2-frame smoke + official Chamfer (needs val CG+HE pairs)
# or: bash src/run_full_smoke.sh (generate_pair_lists + pre-flight checks)

export CG_LIST=$GC2026_ROOT/data/processed/all_cg_only_cgv2.txt
bash src/run.sh
```

Output: `output/submission_candidate_frame_gate_v2/`

**After full inference** (manifest + reproduce README **14.870 mm** on val565):

```bash
bash src/post_submission_candidate.sh
# equivalent to manifest + run_eval.sh; or separately:
#   bash src/download_metric.sh
#   OUT_DIR=... bash src/run_eval.sh
```

`evaluate_uvg.py` uses Metric alignment matrices and official Chamfer. `GATE_JSON` in `common.sh` prefers `data/processed/runtime_gate.json` (same as inference).

`RUN_SMOKE_EVAL=0` skips evaluation during smoke (ENH count check only).

## 7. PD-LTS fine-tuning (optional)

Training scale adapts to the number of train pairs (`patches_per_epoch` auto-scaled).

```bash
bash src/setup_pdlts_train.sh
bash src/run_pdlts_finetune_uvg.sh smoke          # 16-frame quick check
GPUS=4 bash src/run_pdlts_finetune_uvg.sh train
```

Outputs go to `output/pdlts_finetune_uvg/run_*/`; subsequent `run.sh` prefers the newest checkpoint.

## 8. Environment variables

| Variable | Description |
|----------|-------------|
| `GC2026_ROOT` | Workspace root (relative or absolute) |
| `CG_LIST` | Full CG list for inference |
| `PDLTS_FINETUNE_CKPT` | Optional: force PD-LTS checkpoint path |
| `RUN_SMOKE_EVAL` | `1` (default): run Chamfer on 2 smoke frames; `0`: skip |
